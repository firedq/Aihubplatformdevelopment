**目标**：超级管理员能够在 AI-Hub 中创建独立的业务应用（如"供应商入驻协同"、"采购多轮报价"），每个应用生成独立入口链接，支持与不同的后台管理系统做 SSO 集成对接。应用是所有后续操作（OA 集成、表单设计、通知、监控）的业务容器——点击进入应用后，才开展各场景的专属配置。

**FRs covered:** FR1, FR2, FR3

**Stories:**
- Story 2.1: 创建 AI-Hub 业务应用
- Story 2.2: 应用独立入口链接与后台系统 SSO 集成
- Story 2.3: 应用列表查看与授权管理

---

## 业务规则

| # | 规则名称 | 条件 | 结果 |
|---|---------|------|------|
| BR-2.1 | 应用名称唯一性 | 同一租户内应用名称重复 | 拒绝创建，返回 `{"code":1002001,"msg":"应用名称已存在"}` |
| BR-2.2 | 应用名称长度限制 | 名称超过 50 字符 | 前端实时提示超限，提交按钮置灰 |
| BR-2.3 | 应用删除前提 | 仅允许删除草稿状态（未发布任何表单）的应用 | 已发布表单的应用不允许直接删除，需先下线所有表单 |
| BR-2.4 | 软删除策略 | 应用删除操作 | 逻辑删除（deleted=1），关联配置数据同步软删除，不物理清除数据 |
| BR-2.5 | 入口链接唯一性 | 每个应用生成一条专属管理入口链接 | 链接格式：`https://admin.aihub.com/app/{appId}`，appId 全局唯一 |
| BR-2.6 | SSO 首次登录自动建账 | 用户通过 SSO 首次登录 AI-Hub | 系统按 SSO 身份自动创建账号，状态为"待分配角色"，功能受限直至超级管理员授权 |
| BR-2.7 | 应用访问权限控制 | 应用配置管理员登录后 | 只展示已被授权访问的应用，未授权应用不可见、直接访问返回 403 |

## 待确认事项

1. **应用数量限制**：单一租户最多允许创建多少个应用？是否按订阅套餐限制？
2. **SSO 配置作用域**：SSO 集成是以"应用"为单位独立配置，还是以"租户"为单位全局共享一套 SSO？
3. **应用删除策略**：已运行中应用（有历史提交数据）是否永久不允许删除，或允许归档后删除？
4. **应用模板**：未来是否需要支持"从应用模板创建"功能（如供应商入驻标准模板）？

### Story 2.1：创建 AI-Hub 业务应用

As a 超级管理员，
I want 创建一个命名的外部协同应用（如"供应商入驻协同"），
So that 业务团队有一个专属的协同空间，后续在其中配置 OA 集成、设计表单、管理通知。

**前提条件：**
- 超级管理员已登录系统
- 系统基础设施已初始化（Story 1.1）

**操作步骤：**
1. 超级管理员进入"应用管理"页面，点击"新建应用"
2. 填写应用名称（必填，≤50 字符）和应用描述（选填）
3. 点击"创建应用"按钮提交
4. 创建成功后，页面跳转至应用详情页或应用工作台

**成功标志：**
- 应用创建成功，API 返回 `{"code": 0, "data": {"appId": "xxx"}}`
- 应用列表中出现新卡片，显示名称、描述、状态（草稿）、创建时间
- 进入应用工作台后，可见 OA 集成配置、表单列表、通知规则、数据统计等模块入口（均为空白待配置状态）

**页面描述：**
- **应用管理首页**：顶部为"新建应用"按钮；主体为应用卡片网格（每张卡片含：应用图标/默认图、应用名称、描述摘要、表单数量、状态标签、最近修改时间）
- **新建应用弹窗**：应用名称输入框（必填，右侧显示字数计数 X/50）、应用描述文本域（选填）、确认/取消按钮

**Acceptance Criteria:**

**Given** 超级管理员在应用管理页面
**When** 填写应用名称（≤50 字符）、应用描述，点击"创建应用"
**Then** 应用创建成功，API 返回 `{"code": 0, "data": {"appId": "xxx"}}`，应用列表中出现新应用卡片，显示名称、描述、状态（草稿）、创建时间

**Given** 超级管理员填写应用名称
**When** 名称超过 50 字符
**Then** 前端实时提示字数超限，提交按钮置灰，无法提交

**Given** 已存在同名应用（同一租户下）
**When** 再次创建同名应用
**Then** 返回业务错误 `{"code": 1002001, "msg": "应用名称已存在"}`

**Given** 应用已创建
**When** 超级管理员编辑应用名称或描述
**Then** 修改保存成功，应用列表和详情页同步更新

**Given** 应用处于草稿状态（未发布任何表单）
**When** 超级管理员删除该应用
**Then** 应用逻辑删除（deleted=1），应用列表不再显示，关联配置数据同步软删除

**异常情况：**
- 如果应用已有发布中的表单，点击删除时系统提示"该应用下存在已发布的表单，请先下线所有表单后再删除"，阻止删除
- 如果网络超时导致创建失败，页面提示"创建失败，请重试"，输入内容保留

### Story 2.2：应用独立入口链接与后台系统 SSO 集成

As a 超级管理员，
I want 为每个应用生成独立的访问入口链接，并配置与企业内部后台管理系统的 SSO 集成，
So that 内部管理人员通过企业已有账号体系直接单点登录进入对应应用，无需二次登录。

**前提条件：**
- 应用已创建（Story 2.1）
- 企业内部 SSO 系统已就绪，管理员已获取 SSO 配置参数（授权端点/Client ID/Client Secret）

**操作步骤：**
1. 超级管理员进入应用详情页，点击"生成入口链接"
2. 系统自动生成该应用的专属管理入口 URL，页面展示链接，支持一键复制
3. 点击"SSO 配置"标签，选择 SSO 协议类型（OAuth 2.0 / SAML 2.0）
4. 填写 SSO 授权端点、客户端 ID、客户端密钥（SAML 配置填写 IdP 元数据 URL）
5. 点击"保存并测试"，系统发起连通性测试
6. 测试通过后，将应用入口链接嵌入企业门户或发送给内部用户

**成功标志：**
- 入口链接生成成功，格式为 `https://admin.aihub.com/app/{appId}`，支持一键复制
- SSO 连通性测试通过，提示"SSO 配置验证成功"
- 内部用户通过企业门户点击入口链接后，无需二次输入账号密码即可进入对应应用

**页面描述：**
- **应用详情/设置页**：分 Tab 展示：基本信息、入口链接（含生成按钮 + 链接展示 + 复制按钮）、SSO 配置（协议类型选择 + 配置表单）
- **SSO 配置区域**：协议选择（Radio：OAuth 2.0 / SAML 2.0）→ 根据选择动态显示对应配置字段；底部"保存并测试"按钮 + 测试结果提示区域

**Acceptance Criteria:**

**Given** 应用已创建
**When** 超级管理员点击"生成入口链接"
**Then** 系统生成该应用的专属管理入口 URL（如 `https://admin.aihub.com/app/{appId}`），链接展示在应用详情页，支持一键复制

**Given** 超级管理员在 SSO 集成配置页
**When** 填写企业内部 SSO 的授权端点、客户端 ID、客户端密钥（OAuth 2.0 或 SAML 2.0），点击"保存并测试"
**Then** 系统发起 SSO 连通性测试，测试通过时提示"SSO 配置验证成功"；失败时展示具体错误信息（连接超时/授权拒绝/参数错误）

**Given** SSO 配置已完成
**When** 内部管理人员通过企业门户点击应用入口链接
**Then** 自动跳转至企业 SSO 认证，认证通过后回调携带用户身份，AI-Hub 颁发管理端 JWT Token，直接进入对应应用工作台

**Given** SSO Token 中的用户身份在 AI-Hub 中未创建账号
**When** 首次通过 SSO 登录
**Then** 系统自动按 SSO 身份信息创建账号，等待超级管理员分配角色

**异常情况：**
- 如果 SSO 测试连接超时（>10s），提示"SSO 服务连接超时，请检查授权端点地址是否正确"
- 如果 Client Secret 填写错误，SSO 测试返回"授权拒绝（401），请检查 Client ID 和 Client Secret"
- 如果 SSO 服务临时不可用，已登录用户的 JWT Token 仍在有效期内，不影响当前使用

### Story 2.3：应用列表查看与授权管理

As a 应用配置管理员，
I want 查看和管理本组织内已授权给我的应用列表，
So that 我能快速进入对应应用的配置工作台，开展 OA 集成、表单设计等操作。

**前提条件：**
- 应用配置管理员已登录系统
- 超级管理员已完成角色分配并关联了至少一个应用（Story 1.4）

**操作步骤：**
1. 应用配置管理员登录后自动进入应用管理首页
2. 查看已授权的应用卡片列表
3. 点击某应用卡片进入该应用的工作台
4. 在工作台中可见 OA 集成、表单设计、通知规则、数据统计等模块

**成功标志：**
- 只展示被授权的应用，未授权应用不可见
- 点击应用进入工作台后，各功能模块入口可见（权限范围内）
- 权限被撤销后刷新页面，对应应用立即从列表消失

**页面描述：**
- **应用列表页**：同超级管理员视图，但内容仅显示授权范围内的应用卡片；超级管理员视图有"新建应用"按钮，应用配置管理员视图无此按钮
- **应用工作台**：左侧导航含：OA 集成配置、表单管理、通知规则、数据统计；右侧为对应功能的内容区域；顶部面包屑：AI-Hub > {应用名称}

**Acceptance Criteria:**

**Given** 应用配置管理员已登录
**When** 进入应用管理首页
**Then** 只展示本租户内已授权给该用户的应用列表，未授权应用不可见；每个应用卡片展示：名称、描述、表单数量、最近修改时间、状态

**Given** 应用配置管理员在应用列表
**When** 点击某应用卡片
**Then** 进入该应用的专属工作台（含 OA 集成配置、表单列表、通知规则、数据统计等模块入口）

**Given** 超级管理员撤销了应用配置管理员对某应用的访问权限
**When** 该管理员刷新应用列表
**Then** 被撤权的应用从列表消失，尝试直接访问该应用 URL 返回 403

**异常情况：**
- 如果应用配置管理员尝试访问工作台中无权限的子功能（如数据管理员专属的接口监控），返回 403，提示"当前角色无此功能权限"
- 如果该用户的所有应用权限均被撤销，应用列表展示空状态提示"暂未获得任何应用访问权限，请联系超级管理员分配"
